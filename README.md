# 坐标转换工具 - GCJ02 与 WGS84

🌍 一个简单易用的网页工具，用于 GPS 坐标系统（WGS84）与中国地图坐标系统（GCJ02）的相互转换。

## 📋 功能特性

- ✅ **双向转换**：支持 WGS84 ↔ GCJ02 的相互转换
- ✅ **实时验证**：输入坐标时进行动态验证
- ✅ **一键复制**：快速复制转换结果到剪贴板
- ✅ **美观界面**：现代化的渐变设计和响应式布局
- ✅ **精确计算**：采用标准的坐标转换算法，精确到 8 位小数
- ✅ **离线使用**：纯前端实现，无需网络连接
- ✅ **移动友好**：完全适配各种屏幕尺寸

## 🗺️ 坐标系统说明

### WGS84
- **全称**：World Geodetic System 1984
- **用途**：GPS 原始坐标系，全球通用
- **使用场景**：GPS 设备、Google Earth 等
- **特点**：国际标准，全球适用

### GCJ02
- **全称**：Guojia Cehui Ju 02（国测据02）
- **用途**：中国地矿部门发布的坐标系
- **使用场景**：高德地图、腾讯地图等国内地图服务
- **特点**：仅在中国大陆地区使用

## 📊 有效范围

- **经度范围**：73.56° ~ 135.05°（中国东西边界）
- **纬度范围**：3.51° ~ 53.56°（中国南北边界）

**超出范围的坐标**将返回原始坐标值，不进行转换。

## 🚀 快速开始

1. 双击 `index.html` 文件，在浏览器中打开

**使用 Python 2：**

```bash

```

## 💻 使用方法

### 转换 WGS84 → GCJ02
1. 在上方输入框中输入 WGS84 经纬度坐标
2. 点击「WGS84 → GCJ02」按钮
3. 查看转换结果

3. 查看转换结果
4. 可点击「复制」按钮快速复制结果

### 键盘快捷键
- 在输入框中按 `Enter` 键可直接执行对应的转换

## 🔧 技术实现

### 转换算法
采用国务院发布的坐标转换规范，通过以下变换公式：

**WGS84 → GCJ02：**
- 计算地理位置偏差
- 应用高斯-克吕格投影变换
- 计算最终偏移量

**GCJ02 → WGS84：**
- 反向应用偏差修正
- 迭代计算逆向变换

### 前端技术
- **HTML5**：页面结构
- **CSS3**：现代样式设计，包括渐变、阴影、Flexbox 等
- **Vanilla JavaScript**：无依赖，纯原生 JavaScript 实现

## 📝 使用示例

**示例 1：北京朝阳区（WGS84）**
- 输入：经度 116.4074, 纬度 39.9042
- 输出（GCJ02）：经度 116.4176, 纬度 39.9143

**示例 2：上海外滩（GCJ02）**
- 输入：经度 121.4897, 纬度 31.2453

## ⚠️ 注意事项

1. **精度问题**：坐标转换可能存在小数点级别的精度损失
2. **中国范围**：仅适用于中国大陆地区，港澳台不适用
3. **动态数据**：某些情况下转换可能会有细微偏差
4. **离线使用**：本工具完全离线运行，无需传输坐标到服务器
5. **隐私保护**：所有数据处理均在本地浏览器完成，不涉及服务器
